What is ROC-AUC?
A score from 0–1. 0.5 means random guessing; 1.0 means perfect. Our model scores 0.82 — well above chance.
What is "Lift"?
If you picked companies randomly, ~8% would do a deal. Our top predictions have a 58% hit rate — that's 7× better.
What does the model predict?
For each ASX company, it outputs a probability (0–100%) that the company will be involved in a merger, acquisition, or buyout in the next fiscal year.

Model Performance

How well does the model actually work? These metrics are all computed out-of-fold — the model never saw the test data during training, so the numbers are honest.

Precision, Recall & F1 at each threshold Precision = of all flagged companies, how many actually did a deal. Recall = of all companies that did a deal, how many did we catch. The F1 curve peaks at the threshold that best balances the two — that's our recommended operating point.
Are the probabilities accurate? If the model says 70%, do ~70% of those companies actually deal? The green bars show reality; the blue bars show what the model predicted. A perfectly calibrated model would have matching bars. After Platt scaling, ours is close.

Calibration — Can You Trust the Probabilities?

A model can be good at ranking companies (high ROC-AUC) but still give unreliable probabilities. "Calibration" measures whether the scores actually mean what they say.

Why this matters for M&A screening

XGBoost, like most tree models, is naturally overconfident — it tends to push scores toward 0 or 1. A raw score of 80% might only correspond to a 45% real deal rate. Platt scaling fixes this by fitting a logistic curve on top of the raw scores. After calibration, if the model says 80%, roughly 80% of companies at that score really do transact — which is what you need to set meaningful screens.

How It Works

A four-phase pipeline transforms raw ASX filings into calibrated deal probabilities.

01
Model upgrade — from Random Forest to XGBoost

The original model used Random Forest with a random 80/20 train-test split. The problem: a random split lets the model train on 2017 data and "predict" 2014 — that's cheating. We replaced it with walk-forward validation (each year's model is trained only on prior years) and upgraded to XGBoost, which handles class imbalance and gives SHAP explanations for every prediction.

XGBoostWalk-forward CV (no lookahead)SHAP explainability
02
Signal enrichment — 3 new feature groups added

The original model used 11 financial ratios (debt, liquidity, profitability etc.). We added three more signal groups: sentiment (does management language in earnings calls sound acquisitive?), network position (is this company already connected to serial acquirers in past deals?), and macro regime (is the broader market environment conducive to deals — low rates, tight spreads, rising equity?). Total: 31 features.

03
Evaluation — beyond the confusion matrix

The original model only printed a confusion matrix. We added: ROC-AUC and PR-AUC curves (PR-AUC matters more here because deals are rare), Expected Calibration Error to check probability reliability, a full threshold sweep to find the best operating point, and a walk-forward backtest that measures portfolio hit rate and alpha vs the ASX 200 index.

🚫

No lookahead bias

Walk-forward validation ensures every fold only trains on data strictly before the test year. No future information leaks into training.

⚖️

Class imbalance handled

Deals are rare (~8% of company-years). XGBoost's scale_pos_weight parameter is set to the no-deal / deal ratio so the model isn't biased toward predicting "no deal" always.

🎯

Calibrated probabilities

Platt scaling is fitted on a held-out 30% split, so the output probabilities are grounded in reality — not just rank scores.
